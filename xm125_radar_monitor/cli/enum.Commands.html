Variants§

§

Status

Check XM125 radar status and initialization state

Shows device status flags, initialization progress, and error conditions. Use this first to verify the device is responding and properly initialized.

§

Info

Get XM125 device information and firmware details

Displays sensor ID, firmware version, application type, and current configuration. Useful for verifying correct firmware is loaded and device capabilities.

§

Distance

Perform distance measurement

Measures distance to objects with high precision. Automatically configures the device for distance detection mode and handles firmware updates if needed.

Fields

§range: Option<String>

Detection range in meters (format: start:end, e.g., 0.1:3.0)

§continuous: bool

Enable continuous monitoring mode

§count: Option<u32>

Number of measurements in continuous mode (omit for infinite)

§interval: u64

Measurement interval in milliseconds for continuous mode

§save_to: Option<String>

Save measurements to CSV file (continuous mode only)

§

Presence

Perform presence detection

Detects motion and presence with configurable range and sensitivity. Automatically configures the device for presence detection mode.

Fields

§range: Option<PresenceRange>

Presence detection range preset

§min_range: Option<f32>

Minimum detection range in meters (custom range)

§max_range: Option<f32>

Maximum detection range in meters (custom range)

§sensitivity: Option<f32>

Detection sensitivity threshold (0.1 = low, 0.5 = medium, 2.0 = high)

§frame_rate: Option<f32>

Frame rate for presence detection in Hz

§profile: ProfileMode

Profile selection mode

§continuous: bool

Enable continuous monitoring mode

§count: Option<u32>

Number of measurements in continuous mode (omit for infinite)

§interval: u64

Measurement interval in milliseconds for continuous mode

§save_to: Option<String>

Save measurements to CSV file (continuous mode only)

§

Firmware

Firmware management commands

Comprehensive firmware operations including checking, updating, verification, and bootloader control. Handles automatic firmware detection and updates.

Fields

§action: FirmwareAction
§

Gpio

GPIO control and testing commands

Direct GPIO control for XM125 hardware management. Provides initialization, status monitoring, and reset control without external script dependencies.

Fields

§action: GpioAction
