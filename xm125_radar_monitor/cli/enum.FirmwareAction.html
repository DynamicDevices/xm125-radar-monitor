Enum FirmwareAction

pub enum FirmwareAction {
    Check,
    Update {
        firmware_type: FirmwareType,
        force: bool,
        verify: bool,
    },
    Verify {
        firmware_type: Option<FirmwareType>,
    },
    Erase {
        confirm: bool,
    },
    Checksum {
        firmware_type: Option<FirmwareType>,
        verbose: bool,
    },
    Bootloader {
        test_mode: bool,
    },
}

Variants§

§

Check

Check current firmware type and version

Displays the currently loaded firmware information including type (distance/presence), version, and compatibility status.

§

Update

Update firmware to match the specified detector mode

Automatically flashes the correct firmware binary for the selected mode. Uses stm32flash and GPIO control for safe firmware updates.

Fields

§firmware_type: FirmwareType

Target firmware type (distance or presence)

§force: bool

Force update even if firmware already matches

§verify: bool

Verify firmware after update (adds delay and may timeout)

§

Verify

Verify firmware integrity using checksums

Compares the loaded firmware against known good checksums to detect corruption or version mismatches.

Fields

§firmware_type: Option<FirmwareType>

Firmware type to verify against

§

Erase

Erase the XM125 chip completely

This will completely erase all firmware from the XM125 module. The module will need to be reprogrammed before it can be used again. Use with caution - this operation cannot be undone.

Fields

§confirm: bool

Confirm the erase operation (required for safety)

§

Checksum

Calculate and display firmware checksums

Calculates MD5 checksums for firmware binary files to verify integrity and compare against known good versions.

Fields

§firmware_type: Option<FirmwareType>

Specific firmware type to checksum (if not specified, shows all)

§verbose: bool

Show detailed information including file paths and sizes

§

Bootloader

Put XM125 module into bootloader mode for firmware programming

Uses GPIO control to reset the module into bootloader mode (I2C address 0x48). This is required for firmware programming with stm32flash.

Fields

§test_mode: bool

Reset to run mode after entering bootloader (for testing)
